# Phân tích lỗ hổng GraphQL Alias Overloading

lỗ hổng bảo mật trong GraphQL API sử dụng kỹ thuật Alias Overloading. Đây là một vấn đề nghiêm trọng có thể dẫn đến tấn công Từ chối dịch vụ (DoS).

## Cách hoạt động của lỗ hổng

GraphQL cho phép sử dụng alias (bí danh) để thực hiện cùng một truy vấn nhiều lần trong một yêu cầu duy nhất. Khi tính năng này bị lạm dụng, kẻ tấn công có thể:

0. có thể bypass cơ chế tối ưu qua cách thay đổi nội dung của 1 alias đánh lừa Server cho rằng là yêu cầu mới và thực hiện.
1. Gửi một truy vấn GraphQL duy nhất nhưng chứa hàng trăm hoặc hàng nghìn alias
2. Mỗi alias sẽ kích hoạt một lần xử lý riêng trên máy chủ
3. Điều này làm cạn kiệt tài nguyên của máy chủ (CPU, bộ nhớ, kết nối DB)

## Ví dụ tấn công

Một request POST đến GraphQL endpoint có thể trông như sau:

```graphql
  POST /query
  ...
  (Body)
{
  alias1: typename { field1, field2, ... }
  alias2: typename { field1, field2, ... }
  alias3: typename { field1, field2, ... }
  ...
  alias50: typename { field1, field2, ... }
}
```

Trong đó:
- `typename` là tên của một loại đối tượng trong schema GraphQL
- Mỗi alias (`alias1`, `alias2`, ...) khiến máy chủ phải xử lý cùng một truy vấn nhiều lần
- Số lượng alias có thể tăng lên đến hàng trăm hoặc nghìn, gây quá tải hệ thống

## Tác động

1. Tiêu tốn tài nguyên CPU và bộ nhớ máy chủ
2. Làm chậm hoặc crash ứng dụng
3. Làm quá tải cơ sở dữ liệu
4. Từ chối dịch vụ cho người dùng hợp pháp
